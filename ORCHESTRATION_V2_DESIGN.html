2The heart of it — who's allowed to do what your rule #5

You approve these three lanes once. After that the system runs without stopping for permission — except the red lane, which always stops. That's how we get robust safety and no human gate-jams.

🟢 Green — autonomous pre-approved · no stop

Reversible, low-risk, well-understood. The AI does it and logs it. You find out in the morning summary, not by being asked.

e.g. display fixes, demo-data corrections, poka-yoke guards, doc updates, deterministic monitors, front-end deploys that pass smoke + gates.

🟡 Amber — do + notify reversible · 1-click undo

Slightly higher impact but fully reversible (git/backup). The AI does it, then flags it prominently on your cockpit with a one-click revert.

e.g. backend changes that pass tests, data migrations with a verified backup, new monitors.

🔴 Red — always ask never pre-authorised

Irreversible or high-stakes. The system stops and waits for your explicit yes — every time, no exceptions, even if it's 3am.

e.g. anything touching money / the Tuppence ledger / live payments · secrets & security · access permissions · deletions · public posts & outreach · schema-destroying changes.
✓ Honors #5 robust safety (red is immovable) + pre-authorised authority (green/amber run free) · #3 every action is logged, not remembered

4The Second Brain — a self-curating library

The library — the memory

Plain Markdown in git (STATUS, CHANGELOG, BACKLOG, memory notes, baselines) — the Karpathy “LLM Wiki” pattern: the AI grows and interlinks it for you. Every model — cloud Claude now, an open model off-grid later — reads the same store.

Why files, not an app: simplicity, off-grid, git = free versioning + the safe copy. Obsidian stays an optional viewer — never a dependency.

✓ #3 memory · #4 independence / off-grid / fewer tools / simplicity
Live Artifact — your cockpit

One living page. "Since last night: 7 fixed, 2 to review, 1 needs you." Visual, reload for fresh, approve the few reds with a tap.

The only thing you open. No hunting screenshot-by-screenshot.

🟢 The Librarian — a green-lane agent your input, answered

Runs on a schedule with zero effort from you: merges duplicates, fixes stale facts, files new notes into PARA, archives what’s done, keeps the index clean. It is find → fix → prevent pointed at the knowledge itself — so the repository curates itself instead of growing unchecked. (Seed block already exists: the consolidate-memory skill.)

✓ resolves “grows unchecked / needs a librarian” · #3 nothing memory-reliant · #4 no effort
