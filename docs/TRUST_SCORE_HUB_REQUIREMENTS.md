# MarketSquare · My Seller Hub — Trust Score Guidance UI
**Version 0.1 · 27 April 2026**
*Standalone UI requirement — applies to marketsquare_admin.html My Seller Hub screen.*

---

## 1 · Vision

Every seller should know at a glance: what their current score is, what tier they are in, and the single next action that will move them forward. The guidance must be specific — not "add more credentials" but "Upload your PPRA certificate → +15 points → moves you to Established tier."

This screen replaces vague progress indicators with a concrete, actionable checklist that updates in real time as credentials are verified.

---

## 4 · Haiko Tip — "Your Best Next Step"

Below the summary panel, a single Haiko-generated card shows the highest-impact uncompleted action:

```
┌─────────────────────────────────────────────────────┐
│  💡 Best next step                                  │
│                                                     │
│  Upload your PPRA registration certificate.         │
│  This single action adds 15 points — enough to     │
│  reach Established tier and unlock blue badge       │
│  visibility for all buyers.                         │
│                                                     │
│  [Upload now →]                                     │
└─────────────────────────────────────────────────────┘
```

Rules for Haiko tip generation:
- Always show the single highest-point uncompleted action
- If the seller is within 5 points of the next tier, prioritise the action that closes that gap
- If all credentials are complete, show: "Your score is maximised. Keep responding to introductions to build your track record."
- Tip updates immediately after any credential is verified

## 6 · Implementation Notes

- All Trust Score data is read from BEA — never stored or computed client-side
- The checklist state (✅ / ⏳ / ❌) is derived from the seller's verified credentials in the `users` table
- Upload buttons link to the existing `POST /listings/photo` or a new `POST /users/{email}/credential` endpoint
- Haiko tip is generated server-side on page load — single API call returning `{tip_text, action_label, action_url, points_available}`
- The checklist is category-aware — BEA returns the correct credential set based on `seller.category`
- Local Market sellers see the Local Market credential set when they have an active Local Market listing
