Prevent Engine — Phase 4

A poka-yoke per fixed defect, and a monitor for every weak point we don't control. It watches; it never changes anything.

🟢 GREEN LANE · deterministic · zero-token · a failure re-enters the pipeline

What it is

Fix closes a bug; Prevent makes sure it stays closed. Each shipped fix becomes a guard — a deterministic assertion that re-flags the defect the instant it regresses. And each thing we depend on but can't control (external image hosts, feeds) gets a gentle monitor. The trick: a guard or monitor failure is written back out as a Detect-shaped finding, so it flows straight into Triage → Fix. The loop closes on itself — the system watches its own past work.
