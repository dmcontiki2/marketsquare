Fix Engine — Phase 3

Consumes the green lane of a triaged run, one item at a time, under the verify-or-revert gates. The end of the Detect → Triage → Fix tissue.

🟢 GREEN LANE · pre-authorised · backup + verify + revert · RED never touched

What it is

Triage governs; Fix acts — but only inside the lane you pre-approved. It takes the green queue from triaged.json, highest priority first, and for each item makes one surgical, verified, reversible change and ships it. Amber is staged for a glance; red is never consumable here. Anything bigger than a surgical change is routed onward, not forced.

How you call it

> Run the Fix engine on the triaged queueharness: fix.py --next · fix.py --ship <id> --detail "…" · fix.py --route <id> --to <ref> · fix.py --fail <id>

How it runs — one item, fully, then the next

1
Pick the top green itemFrom triaged.json, lane=green & status=queued, highest priority first. Amber/red are skipped (amber stays staged; red waits for you).
2
Back up firstA timestamped server backup of every target file before a byte changes — rollback is always one copy away.
3
Surgical editOne well-understood change via a bash-python str.replace driver (never the Edit/Write tools on big files), asserting the old text matches exactly once.
4
Verify — or revertnode --check / ast.parse, a whole-file diff that is only the intended change, then smoke_test.py must stay all-green. Any failure → revert, mark failed, never deploy a red smoke.
5
Ship (green is pre-authorised)Deploy (scp + cache-bump + chmod 644 + Cloudflare purge), re-verify live, refresh the FEA baseline. Green ships without stopping to ask — that authority was granted once, in Phase 0.
6
Record & drain the boardFlip the item to shipped/routed/failed in triaged.json, append fix_results.json, regenerate the board so the queue visibly empties.

The lane gate at the moment of action

🟢 Green

verified → shipped unattended. Reversible, logged.

🟡 Amber

done + staged with a 1-click undo. Never silently shipped.

🔴 Red

never touched. Stays for the human — every time, no exceptions.

Guardrails

🔁
One item per runSmall blast radius. Each fix is isolated, verified and recorded on its own, so a failure never cascades.
↩️
Verify-or-revertBackup before, verify after, revert on any failure. A red smoke is never deployed.
↪️
Never force a non-surgical changeIf the right fix is bigger than a surgical edit (a refactor, a data/storage migration), Fix routes it to the proper track instead of hacking it — it never makes things worse.
