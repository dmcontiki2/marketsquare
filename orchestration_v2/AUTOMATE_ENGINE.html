The cutover — the one deliberate switch left

Gated on parity, by design. The Phase-0 plan is explicit: run v2 beside the old loop, compare for a night or two, then cut over. So none of the steps below fire on their own — they wait for your “go”, and every one is reversible.
  1. Watch a parity night. Let the 03:50 conductor run once or twice beside the old loop; compare its report to the old loop’s report.json. It should see the same or more, for no tokens.
  2. Turn on the v2 Fixer (Sonnet checkpoint). A small scheduled task that reads the green work order from orchestrator_v2_report.json, makes the one surgical fix under the lane gates (verify-or-revert + smoke), ships green / stages amber·red, and records — the same discipline used by hand in Phase 3.
  3. Retire the old loop. Disable the three old Claude tasks — trustsquare-orch-sensor, -fixer, -orchestrator — in one move each (reversible: re-enable any time).
  4. Flip the conductor to live. Change the cron from orchestrator_v2.py to orchestrator_v2.py --live. The deterministic sensor + conductor + the v2 Fixer are now the loop; the old patch-pile is gone.
Fully reversible at every step. Re-enable the old tasks, drop the v2 Fixer, or remove the conductor’s cron line — nothing here deletes data or deploys code. The old loop keeps running untouched until step 3.

Why this finishes the rewiring

With Automate live, all five stages run as one self-healing loop on the box — Detect → Triage → Fix → Prevent — surfacing only amber/red and a one-line summary to your cockpit, for essentially zero tokens. The clean rewiring is complete, and the old patched loop (the one that kept truncating BACKLOG) is retired by a controlled switch, not a morph. The system now maintains itself.
