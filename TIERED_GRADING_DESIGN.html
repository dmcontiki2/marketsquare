Evidence-Based Tiered Grading

Condition as a verification ladder, not a seller's claim · 3 June 2026 · cards first, cars to follow · ties to Trust Score + the Audit Function

The principle (David, 3 June). A seller cannot grade their own item — they lack the standing and often the skill. So TrustSquare derives condition from evidence, and lets that evidence get stronger in tiers. The first grade the app accepts is an AI read of the photos; better grades come from verification. This is the marketplace's whole premise applied to condition: trust from evidence, not identity.

It is almost certainly part of the Audit Function — a progressive, auditable provenance ladder — and therefore likely patent-relevant. Design the two as one idea.

A listing always sits at a tier, and the badge says which. A buyer instantly knows whether "Near Mint" is an AI guess from a photo or a PSA 9 on a cert — that distinction is the product.

How Tier 1 (AI grading) works

At listing time the seller supplies the required photos. A vision model reads them and returns a structured verdict: a grade from the agreed vocabulary, a confidence (0–1), and a one-line rationale ("soft corners + light edge whitening → Lightly Played"). The grade is stored, displayed with its tier badge and the honesty label, and feeds the Trust Score. Low-confidence reads are flagged for verification rather than shown as fact.

Honesty rail. An AI photo grade is indicative. It must never be presented as a certified grade, and the UI must always show the tier so an AI read is never mistaken for a verified one. Over-claiming condition is a trust- and liability-risk we don't take.

Two vocabularies, one ladder

Raw items use descriptive grades the eye/AI can judge — TCGplayer's Near Mint / Lightly Played / Moderately Played / Heavily Played / Damaged. Certified items carry a numeric grade — PSA/BGS/CGC 1–10. The ladder spans both: Tier 1 speaks descriptive, Tier 2 adds numeric + provenance. The badge maps between them so the feed stays scannable.

Cars — the same idea, different evidence

Tier 1 is looks (photos) plus logbook completeness — a present, complete logbook is itself a trust signal. Tier 2 layers the documented record: full service history, DEKRA / roadworthy certificate, accident/finance checks (all strong, verifiable SA signals). Tier 3 is the expert drive — irreplaceable, because a car's true condition is heard and felt, not photographed. Tier 3 is a premium/partner verification, not a default.

Where it plugs in

  • Trust Score — grade tier is a Trust input: a verified Tier 2/3 listing should score and rank above an unverified Tier 1.
  • Audit Function — the tier history (AI grade → cert → expert) is an auditable provenance trail. This is the shared spine; build them together.
  • Existing AI vision — reuses the vision pipeline already in the BEA (KYC/safety vision), so this is an extension, not a new stack.
  • Anonymity (A2) — graders see the item, never the seller; photos must carry no identity (no faces, plates, documents). Same rule the listing photos already follow.
Cost-model impact. Tier 1 is one vision call per listing (or per re-grade). Negligible per item, real at volume — so it belongs in the free/paid AI-feature tiering (Tier 1 free as a trust-builder; verification flows can be paid/partner). Flag in the cost model when this goes to volume.

FieldMeaning

cert_grade / cert_id / cert_bodyTier-2 certificate: numeric grade, cert number, PSA/BGS/CGC

The existing condition column (the seller/back-filled descriptive value) stays as-is. ai_grade is separate, so nothing is destroyed and the two can be compared during review.
